# 🎯 '꿈꾸는 내일' 플래너 배포 완벽 가이드

가장 정확하고 실패 없는 두 가지 배포 방법을 순서대로 안내해 드립니다.

---

## 방법 1: 가장 추천하는 방법 (GitHub Desktop + Vercel)
명령어를 입력할 필요 없이 클릭만으로 배포하는 방법입니다.

### 1단계: GitHub에 코드 올리기
1. [GitHub Desktop](https://desktop.github.com/)을 설치하고 로그인합니다.
2. 상단 메뉴에서 **File > Add local repository**를 클릭합니다.
3. 현재 프로젝트 폴더(`todo-app`)를 선택합니다.
4. "Initialize this repository with a README"를 체크하지 **않고** 추가합니다.
5. 하단의 **Summary** 칸에 "Initial commit"이라고 적고 **Commit to main**을 누릅니다.
6. 상단의 **Publish repository** 버튼을 눌러 GitHub 웹사이트로 코드를 보냅니다.

### 2단계: Vercel과 연결하기
1. [Vercel](https://vercel.com/)에 접속하여 GitHub 계정으로 로그인합니다.
2. **Add New > Project**를 누릅니다.
3. 방금 올린 `todo-app` 저장소 옆의 **Import**를 누릅니다.
4. **Build and Output Settings**는 건드릴 필요 없이 **Deploy** 버튼을 누릅니다.
5. 약 1분 뒤, `https://todo-app-username.vercel.app` 같은 링크가 생성됩니다!

---

## 방법 2: 가장 빠른 방법 (드래그 앤 드롭)
GitHub를 거치지 않고 바로 파일을 올리는 방법입니다.

1. 현재 폴더에서 `npm run build`를 실행합니다. (제가 이미 실행해 두었습니다!)
2. 폴더 안에 생긴 **`dist`** 폴더를 확인합니다. 이 폴더가 실제 서비스될 파일들입니다.
3. [Vercel Deployment Dashboard](https://vercel.com/import/deploy)에 접속합니다.
4. 화면 중앙의 드래그 앤 드롭 영역에 **`dist`** 폴더를 통째로 끌어다 놓습니다.
5. 즉시 배포가 시작되며, 완료 즉시 접속 가능한 주소가 나옵니다.

---

### 💡 팁
- **수정사항 적용**: 코드를 수정했다면 다시 `npm run build`를 하고 `dist` 폴더를 새로 올리거나(방법 2), GitHub Desktop에서 **Commit & Push**(방법 1)를 하면 자동으로 업데이트됩니다.
- **주소 설정**: Vercel 설정에서 원하는 도메인 주소로 변경할 수 있습니다.

이제 이 순서대로 따라하시면 나만의 멋진 플래너를 인터넷에서 만나실 수 있습니다! 🚀🌙
